This is my fork of [Christian Lempa's homelab documentation](https://github.com/ChristianLempa/homelab) — a collection of notes, setups, and configuration files covering infrastructure, applications, and networking. Christian publishes it as a free companion to his IT and homelab YouTube content, and I keep it forked as a reference for my own Proxmox cluster and self-hosted services.

## The concepts

Rather than a single deployable project, this is a documentation repository: a curated set of working configurations and notes for the tools a serious homelab tends to accumulate. The value is in seeing how someone who does this professionally structures and documents their setups, which makes it a good source of patterns to adapt rather than copy wholesale. The upstream author notes that products change over time, so any given config is a snapshot worth understanding before applying.

## How to use it

You browse to the tool or topic you care about, read the notes, and adapt the configuration to your environment. It pairs naturally with the related upstream repos — dotfiles, boilerplates, and cheat-sheets — for a fuller picture.

## Running it

There's nothing to deploy directly; it's reference material. Fork it, take what's useful, and use it as documentation for your own lab.
